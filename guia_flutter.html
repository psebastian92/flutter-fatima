¿Qué es Flutter y por qué lo usamos?
Contexto antes de arrancar
📱
Flutter es una herramienta de Google para hacer apps de celular. Se programa en un lenguaje que se llama Dart — muy parecido a Java, que ya conocen. Lo escriben en VS Code y con un comando se convierte en una APK para instalar en Android.
💡
Nuestra app no necesita mucha pantalla — es una app que corre en segundo plano, escucha el Bluetooth del ESP32 y abre Maps cuando llega la señal. La interfaz es solo una pantalla simple que dice "Conectado" o "Esperando…"
2
Instalar Flutter y configurar VS Code
Una sola vez por computadora
1
Descargar Flutter SDK

Ir a flutter.dev → clic en "Get started" → descargar para Windows. Descomprimir en C:\flutter (no en Archivos de programa).

⚠️
La carpeta NO puede tener espacios ni caracteres especiales en la ruta. C:\flutter funciona. C:\Mis Archivos\flutter no.
2
Agregar Flutter al PATH del sistema

Buscar "Variables de entorno" en Windows → PATH → Nuevo → pegar C:\flutter\bin → Aceptar.

3
Instalar Java JDK (si no lo tienen)

Flutter necesita Java para compilar la APK. Descargar JDK 17 desde adoptium.net. Si ya tienen Java de los proyectos con Tomcat, puede servir.

java -version
4
Instalar Android SDK (sin Android Studio)

Descargar solo las "command-line tools" desde developer.android.com/studio#command-tools. Descomprimir en C:\android-sdk. Luego en la terminal:

sdkmanager "platform-tools" "platforms;android-33" "build-tools;33.0.0"
5
Instalar extensiones en VS Code

Abrir VS Code → Extensions (Ctrl+Shift+X) → buscar e instalar:

📦
Flutter (de Dart Code) — instala Dart automáticamente también
6
Verificar que todo esté bien

Abrir una terminal nueva en VS Code y correr:

flutter doctor
✅
Tiene que aparecer tilde verde en "Flutter", "Android toolchain" y "VS Code". Si algo tiene ✗, el comando explica qué falta.
7
Aceptar licencias de Android
flutter doctor --android-licenses

Tipear "y" y Enter en cada pregunta.

4
Agregar los paquetes necesarios
Como agregar librerías en Java, pero más fácil
💡
En Flutter los paquetes se declaran en pubspec.yaml — es como el pom.xml de Maven pero mucho más simple. Después un comando los descarga solos.

Abrir pubspec.yaml y reemplazar la sección dependencies: con esto:

pubspec.yaml
dependencies:
  flutter:
    sdk: flutter
  flutter_blue_plus: ^1.31.0 # Bluetooth BLE
  http: ^1.2.0 # Para hablar con el Servlet
  url_launcher: ^6.2.0 # Abrir Maps, llamar, SMS
  flutter_tts: ^4.0.2 # Hacer hablar al celular

Guardar y ejecutar:

flutter pub get

También hay que declarar permisos en android/app/src/main/AndroidManifest.xml — agregar antes de <application>:

AndroidManifest.xml — permisos
<!-- Bluetooth -->
<uses-permission android:name="android.permission.BLUETOOTH"/>
<uses-permission android:name="android.permission.BLUETOOTH_ADMIN"/>
<uses-permission android:name="android.permission.BLUETOOTH_SCAN"/>
<uses-permission android:name="android.permission.BLUETOOTH_CONNECT"/>
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
<!-- Internet (para hablar con el Servlet) -->
<uses-permission android:name="android.permission.INTERNET"/>
5
El código completo de la app
Todo en lib/main.dart
📋
Reemplazar todo el contenido de lib/main.dart con este código. Cambiar la URL del Servlet por la dirección real de su servidor.
lib/main.dart
import 'package:flutter/material.dart';
import 'package:flutter_blue_plus/flutter_blue_plus.dart';
import 'package:http/http.dart' as http;
import 'package:url_launcher/url_launcher.dart';
import 'package:flutter_tts/flutter_tts.dart';

// ── CONFIGURACIÓN ──────────────────────────────────
const SERVLET_URL = 'http://192.168.1.100:8080/ProyectoBotonera/ConfigServlet';
const ESP32_NOMBRE = 'Botonera'; // nombre BLE del ESP32
// ───────────────────────────────────────────────────

void main() => runApp(const AppBotonera());

class AppBotonera extends StatelessWidget {
  const AppBotonera({super.key});
  @override
  Widget build(BuildContext context) => MaterialApp(
    title: 'Botonera',
    theme: ThemeData(colorScheme: ColorScheme.dark()),
    home: const PantallaBotonera(),
  );
}

class PantallaBotonera extends StatefulWidget {
  const PantallaBotonera({super.key});
  @override
  State<PantallaBotonera> createState() => _Estado();
}

class _Estado extends State<PantallaBotonera> {

  // Guarda la config: "BTN1" → ("maps", "Hospital Argerich...")
  Map<String, List<String>> config = {};
  String estado = 'Iniciando...';
  final tts = FlutterTts();

  @override
  void initState() {
    super.initState();
    tts.setLanguage('es-AR');
    descargarConfig(); // 1° baja la config del Servlet
    conectarBluetooth(); // 2° se conecta al ESP32
  }

  // ── PASO 1: Bajar config del Servlet ────────────────
  Future<void> descargarConfig() async {
    setState(() => estado = 'Descargando configuración...');
    try {
      final respuesta = await http.get(Uri.parse(SERVLET_URL));
      // El Servlet responde líneas como: "1|maps|Hospital Argerich"
      for (var linea in respuesta.body.trim().split('\n')) {
        final partes = linea.split('|');
        if (partes.length == 3) {
          config['BTN${partes[0]}'] = [partes[1], partes[2]];
        }
      }
      setState(() => estado = 'Config lista. Buscando botonera...');
    } catch (e) {
      setState(() => estado = 'Error al bajar config: $e');
    }
  }

  // ── PASO 2: Conectarse al ESP32 por Bluetooth ────────
  Future<void> conectarBluetooth() async {
    setState(() => estado = 'Buscando botonera por Bluetooth...');
    FlutterBluePlus.startScan(timeout: const Duration(seconds: 10));

    FlutterBluePlus.scanResults.listen((results) {
      for (var r in results) {
        if (r.device.platformName == ESP32_NOMBRE) {
          FlutterBluePlus.stopScan();
          _conectarDispositivo(r.device);
        }
      }
    });
  }

  Future<void> _conectarDispositivo(BluetoothDevice dispositivo) async {
    await dispositivo.connect();
    setState(() => estado = '✅ Botonera conectada. Listo.');

    final servicios = await dispositivo.discoverServices();
    for (var servicio in servicios) {
      for (var caracteristica in servicio.characteristics) {
        if (caracteristica.properties.notify) {
          await caracteristica.setNotifyValue(true);
          // Cada vez que el ESP32 manda algo, se ejecuta esto:
          caracteristica.lastValueStream.listen((bytes) {
            final mensaje = String.fromCharCodes(bytes).trim();
            ejecutarAccion(mensaje); // ej: "BTN1"
          });
        }
      }
    }
  }

  // ── PASO 3: Ejecutar la acción del botón ─────────────
  void ejecutarAccion(String boton) {
    final datos = config[boton];
    if (datos == null) return; // botón sin configurar

    final tipo = datos[0]; // "maps", "llamar" o "mensaje"
    final valor = datos[1]; // la dirección, teléfono o texto

    switch (tipo) {
      case 'maps':
        tts.speak('Navegando a $valor');
        abrirMaps(valor);
      case 'llamar':
        tts.speak('Llamando');
        llamar(valor);
      case 'mensaje':
        tts.speak('Enviando mensaje');
        enviarMensaje(valor);
    }
  }

  void abrirMaps(String direccion) {
    final uri = Uri.parse('geo:0,0?q=${Uri.encodeComponent(direccion)}');
    launchUrl(uri);
  }

  void llamar(String telefono) {
    final uri = Uri.parse('tel:$telefono');
    launchUrl(uri);
  }

  void enviarMensaje(String datos) {
    // formato: "numero|texto del mensaje"
    final partes = datos.split('|');
    final uri = Uri.parse('sms:${partes[0]}?body=${Uri.encodeComponent(partes[1])}');
    launchUrl(uri);
  }

  // ── PANTALLA (simple, solo muestra el estado) ────────
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      backgroundColor: const Color(0xFF0D1117),
      body: Center(
        child: Column(mainAxisAlignment: MainAxisAlignment.center, children: [
          const Text('🔲', style: TextStyle(fontSize: 64)),
          const SizedBox(height: 16),
          const Text('Botonera', style: TextStyle(color: Colors.white, fontSize: 22, fontWeight: FontWeight.bold)),
          const SizedBox(height: 8),
          Text(estado, style: const TextStyle(color: Colors.grey, fontSize: 14), textAlign: TextAlign.center),
          const SizedBox(height: 24),
          ElevatedButton(
            onPressed: () { descargarConfig(); conectarBluetooth(); },
            child: const Text('Reconectar'),
          ),
        ]),
      ),
    );
  }
}
6
Probar en el celular
Sin generar APK todavía
1
Activar opciones de desarrollador en el celular

Ajustes → Acerca del teléfono → tocar "Número de compilación" 7 veces. Aparece "Opciones de desarrollador".

2
Activar "Depuración USB"

Ajustes → Opciones de desarrollador → activar "Depuración USB". Conectar el celular a la PC con cable USB.

3
Correr la app directo en el celular
flutter run
✅
Flutter detecta el celular, compila y lo instala automáticamente. Se ve en tiempo real en el celu.
7
Generar la APK
Para instalar en cualquier celular sin cable
1
Compilar la APK
flutter build apk --release

Tarda unos minutos la primera vez.

2
Encontrar el archivo generado

La APK queda en:

build/app/outputs/flutter-apk/app-release.apk
3
Instalar en el celular

Mandar el archivo por WhatsApp, Drive o cable USB. Abrirlo en el celular. Android va a pedir permiso para instalar apps de orígenes desconocidos — aceptar.

💡
Para instalar desde fuera de la Play Store: Ajustes → Seguridad → activar "Instalar apps desconocidas" para el navegador o gestor de archivos que usen.
