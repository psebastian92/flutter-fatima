¿Qué es Flutter?

Flutter es un framework de Google que permite crear apps para Android (y también iOS, web y escritorio) escribiendo un solo código. El lenguaje que usa se llama Dart. No hace falta aprenderlo antes de empezar — lo vas incorporando mientras programás.

Flutter
El framework
El conjunto de herramientas que nos da los componentes visuales, la lógica de la app y el compilador.
Dart
El lenguaje
El lenguaje de programación que usa Flutter. Es similar a Java o JavaScript, fácil de aprender.
VS Code
El editor
Donde escribimos el código. Tiene extensiones específicas para Flutter que ayudan mucho.
APK
El resultado
El archivo que instala la app en cualquier celular Android, como los que bajás de la Play Store.
Instalación del entorno
1
Instalar Flutter SDK
El paquete principal — contiene el compilador y todas las herramientas base
1
Descargar el ZIP de Flutter

Entrar a flutter.dev → Get started → Windows. Bajar el ZIP de la versión Stable (la más reciente y probada).

ℹ️
La versión Stable es la que funciona mejor para empezar. Las versiones Beta o Dev pueden tener errores.
2
Descomprimir en C:\flutter

Click derecho sobre el ZIP → Extraer aquí. Mover la carpeta resultante a la raíz del disco:

C:\flutter
⛔
No usar estas ubicaciones:
❌ C:\Archivos de programa\flutter — tiene espacio en el nombre
❌ C:\Usuarios\Sebastián\flutter — tiene tilde
❌ C:\Mi Carpeta\flutter — tiene espacio

El compilador de Android falla silenciosamente con espacios y caracteres especiales en el path.
3
Agregar Flutter al PATH del sistema

El PATH le dice a Windows dónde buscar programas. Hay que agregarlo para poder escribir flutter en cualquier terminal.

Seguir estos pasos:

  1. Tecla Windows → buscar "variables de entorno" → abrir
  2. Clic en Variables de entorno...
  3. En la sección de abajo (Variables del sistema), seleccionar Path → Editar
  4. Clic en Nuevo → escribir C:\flutter\bin
  5. Aceptar en todas las ventanas
4
Verificar que Flutter quedó instalado

Abrir una terminal nueva (importante: nueva, no una que ya estaba abierta) y escribir:

flutter --version

Debería aparecer algo como:

Flutter 3.x.x • channel stable Dart 3.x.x
⚠️
Si dice "flutter no se reconoce como comando", el PATH no quedó bien. Cerrar y volver a abrir la terminal. Si sigue fallando, revisar el paso 3.
2
Instalar Java y Android SDK
Flutter los necesita para compilar apps Android
1
Instalar Java JDK 17

Entrar a adoptium.net → bajar OpenJDK 17, Windows x64, instalador .msi. Ejecutarlo y seguir los pasos (todo por defecto está bien).

ℹ️
Java es necesario porque el compilador de Android está escrito en Java. No vamos a programar en Java — solo necesitamos que esté instalado.
2
Instalar Android Studio

Entrar a developer.android.com/studio y bajar el instalador. Durante la instalación, dejar todo por defecto. Android Studio incluye el Android SDK, que es el conjunto de herramientas para compilar apps Android.

💡
No vamos a usar Android Studio para programar (eso lo hace VS Code). Lo instalamos solo para que el SDK quede disponible en la PC.
3
Aceptar las licencias de Android

Flutter necesita que aceptemos los términos de uso del SDK de Android. Abrir la terminal y correr:

flutter doctor --android-licenses

Va a hacer varias preguntas, responder y + Enter a cada una.

4
Verificar que todo esté instalado correctamente flutter doctor

Aparece una lista de chequeos. Los importantes son:

[✓] Flutter (Channel stable) [✓] Android toolchain [✓] Android Studio [!] VS Code (lo instalamos en el paso 3)
ℹ️
Si dice que falta el NDK: abrir Android Studio → SDK Manager (ícono de la barra de herramientas) → pestaña SDK Tools → marcar NDK (Side by side) → Apply.
3
Configurar VS Code
El editor donde vamos a escribir el código
1
Instalar la extensión de Flutter

En VS Code: abrir el panel de extensiones con Ctrl+Shift+X → buscar Flutter → instalar la publicada por Dart Code.

Esto también instala la extensión de Dart automáticamente.

✅
Con estas dos extensiones, VS Code va a resaltar el código, sugerir completado automático, mostrar errores en tiempo real y tener botones para correr la app.
2
Confirmar que VS Code encontró Flutter

Abrir la paleta de comandos con Ctrl+Shift+P → escribir Flutter: New Project. Si la opción aparece en la lista, todo está correctamente configurado.

Tu primer proyecto
4
Crear el proyecto
Un proyecto Flutter nuevo trae una app de ejemplo funcionando desde el primer momento
1
Elegir dónde guardar los proyectos

Crear una carpeta en una ruta sin tildes, ñ ni espacios. Por ejemplo:

C:\proyectos
2
Crear el proyecto desde la terminal de VS Code

Abrir la terminal integrada de VS Code con Ctrl+`. Ir a la carpeta de proyectos y ejecutar:

cd C:\proyectos
flutter create mi_primera_app
💡
El nombre del proyecto solo puede tener letras minúsculas, números y guiones bajos. No puede empezar con número.
3
Abrir el proyecto en VS Code cd mi_primera_app
code .

El punto (.) le dice a VS Code que abra la carpeta actual.

5
Entender la estructura del proyecto
Qué es cada carpeta y qué archivos vas a editar
1
Mapa del proyecto
mi_primera_app/ ├── android/ │ └── app/src/main/ │ └── AndroidManifest.xml ← permisos de la app │ ├── lib/ │ └── main.dart ← TODO el código de la app va acá │ ├── pubspec.yaml ← nombre, versión y paquetes externos │ └── ios/ web/ linux/ ... ← otras plataformas, ignorar por ahora
2
Los tres archivos clave
ArchivoQué hace¿Cuándo se toca?
lib/main.dart El código de la app: pantallas, botones, lógica Todo el tiempo
pubspec.yaml Declara los paquetes externos que usa la app Cuando se agrega una librería nueva
AndroidManifest.xml Permisos que pide la app (internet, cámara, etc.) Cuando la app necesita acceder a algo del sistema
3
El main.dart que viene de ejemplo

Flutter crea una app de contador funcionando. Es útil para entender la estructura básica antes de empezar a escribir código propio.

🎯
Antes de borrar el código de ejemplo, correrlo en el celular (paso 7) para confirmar que el entorno funciona bien.
6
Agregar paquetes externos
Librerías listas para usar que le suman funciones a la app
1
¿Qué es un paquete?

Un paquete es código que alguien más escribió y publicó para que cualquiera lo pueda usar. En vez de programar todo desde cero, se declara el paquete y Flutter lo descarga automáticamente.

El repositorio oficial de paquetes Flutter es pub.dev — ahí se busca por nombre y se ven instrucciones de uso.

2
Declarar paquetes en pubspec.yaml

Agregar los paquetes bajo la sección dependencies. El número después del ^ es la versión mínima:

pubspec.yaml
dependencies: flutter: sdk: flutter http: ^1.2.1 # hacer pedidos a un servidor url_launcher: ^6.2.5 # abrir URLs, llamadas, WhatsApp shared_preferences: ^2.2.2 # guardar datos en el celular
3
Descargar los paquetes

Después de editar pubspec.yaml, ejecutar en la terminal:

flutter pub get
✅
VS Code suele hacerlo automáticamente al guardar el archivo. Si no, correrlo a mano desde la terminal.
4
Usar el paquete en el código

Al principio de main.dart, importar el paquete:

lib/main.dart
import 'package:http/http.dart' as http; import 'package:url_launcher/url_launcher.dart';
Probar y distribuir
7
Probar la app en el celular
Ver la app corriendo en un dispositivo real, en tiempo real
1
Activar las Opciones de Desarrollador en el celular

Ir a Ajustes → Acerca del teléfono → tocar 7 veces seguidas sobre Número de compilación.

Aparece el mensaje "Ya sos desarrollador". Volver a Ajustes — ahora aparece una nueva opción: Opciones para desarrolladores.

2
Activar la Depuración USB

Entrar a Opciones para desarrolladores → activar Depuración por USB.

Conectar el celular a la PC con el cable. El celular va a pedir confirmación — tocar Permitir.

ℹ️
En algunos celulares "Número de compilación" está en Ajustes → Información del software o Ajustes → Sistema → Acerca del teléfono. El lugar varía según la marca.
3
Verificar que Flutter detecta el celular flutter devices

Debe aparecer el celular en la lista. Si no aparece, probar con otro cable USB (algunos cables solo cargan y no transmiten datos).

4
Correr la app flutter run

Flutter compila e instala la app en el celular automáticamente. La primera vez tarda más (descarga dependencias). Las siguientes veces es más rápido.

🔥
Mientras la app corre, desde la terminal se puede:
r → recargar cambios sin reiniciar la app (hot reload)
R → reiniciar la app completamente (hot restart)
q → cerrar
5
Ver los mensajes de debug

Los print('algo') que escribas en el código aparecen en la terminal de VS Code mientras la app corre. Es la forma más simple de ver qué está pasando.

8
Generar la APK
Un archivo que se puede instalar en cualquier celular Android sin necesitar la PC
1
Compilar la APK flutter build apk --release

Tarda unos minutos. El flag --release genera una versión optimizada y más liviana (sin herramientas de debug).

2
Encontrar el archivo generado

Flutter genera la APK automáticamente en:

build/app/outputs/flutter-apk/app-release.apk
3
Instalar en otro celular

Mandar el archivo por WhatsApp, Google Drive o Bluetooth. Abrirlo en el celular. Android va a mostrar una advertencia sobre aplicaciones de orígenes desconocidos.

💡
Para poder instalar la APK hay que dar permiso una vez:
Ajustes → Seguridad → Instalar apps desconocidas → activar para el gestor de archivos o el navegador que estén usando.
